🎯 QA Benefits

⏱️

Reduce Test Authoring Time

Instead of manually capturing API calls with proxy tools, the extension auto-correlates UI actions to network requests. QA writes tests 3-5x faster.

🔗

Automatic Correlation

Smart algorithm links UI clicks to API calls using timing, request importance, and keyword matching. No manual pairing needed.

✅

Built-in Assertions

Add UI and API assertions during review. Export includes ready-to-run TestNG assertions, not just comments.

🔄

Data-Driven Testing

Extract variables from responses and reuse them in subsequent steps. Supports login → token → authenticated requests.

🧹

Smart Noise Filtering

Automatically classifies and filters analytics, auth refresh, and polling requests. Keeps only business-relevant APIs.

📊

Review Before Export

QA can verify correlation accuracy, toggle API relevance, add notes, and adjust confidence scores before generating tests.

📤 Export Example (Selenium + Rest Assured)

// Generated by FlowTrace QA
@Test
public void testLoginFlow() {
    // Step 1: Login Action
    driver.findElement(By.cssSelector("#username")).sendKeys("testuser");
    driver.findElement(By.cssSelector("#password")).sendKeys("password123");
    driver.findElement(By.cssSelector("button[type='submit']")).click();
    
    // Step 1: API Verification
    ValidatableResponse response = given()
        .header("Content-Type", "application/json")
        .body("{\"username\":\"testuser\",\"password\":\"password123\"}")
    .when()
        .post("https://api.example.com/login")
    .then()
        .statusCode(200)
        .body("token", notNullValue());
    
    // Step 2: Navigate to Dashboard
    driver.findElement(By.cssSelector(".nav-dashboard")).click();
    
    // Assertions
    assertThat(driver.findElement(By.cssSelector(".user-name")).getText())
        .isEqualTo("Welcome, testuser");
}

🔄 QA Workflow with FlowTrace

1. Record → Navigate through the app, perform user actions

2. Review → Verify correlations, toggle API relevance, add assertions

3. Export → Generate Selenium/Rest Assured code or JSON

4. Execute → Run generated tests in CI/CD pipeline

5. Maintain → Re-record only changed flows, not entire test suites
